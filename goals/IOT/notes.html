Although Arduino is a great choice for microcontroller-based projects, I decided to start my IoT journey with the Raspberry Pi because of my background in Python programming from university. Since the Pi supports full Python scripting out of the box, it made sense for me to build on what I already know instead of starting from scratch with Arduino's C/C++ environment.

The Raspberry Pi also gives me access to a full Linux environment, GPIO pins, internet connectivity, and the flexibility to build full-stack IoT systems ‚Äî from hardware control to backend logic and eventually frontend dashboards.

My journey into Full Stack IoT began this summer with a strong motivation to master hardware and software integration before university resumes. Since I already had a background in Python from my university courses, I decided to go with the Raspberry Pi instead of Arduino. Arduino is great for microcontroller projects, but the Pi gave me the flexibility of using Python, working within a Linux environment, and building more complex IoT systems.

I had used a normal phone charger instead of the recommended Pi power supply, I learned that the Pi requires a stable input of 100 - 240V and ouput of 5.1V power supply with 2.5A of current (depending on the model), while most phone chargers only supply around 1A‚Äì2A and can‚Äôt consistently handle the Pi‚Äôs power demands ‚Äî especially when using peripherals like a keyboard, mouse, or display. üí° Project 1 ‚Äì Blinking Two LEDs

This was my first hands-on GPIO project on the Raspberry Pi. I used Python and the RPi.GPIO library to blink two LEDs in alternating patterns. The goal was to get comfortable working with basic circuit components like LEDs, resistors, breadboards, and GPIO pins.

I connected two LEDs to GPIO pins 2 and 4 and wrote a simple loop in Python to turn them on and off at 0.5-second intervals. Although the video I recorded was just 6 seconds long, it was enough to confirm that the logic worked. Seeing the LEDs blink for the first time was a huge motivational boost.

üß† Code & Logic Explanation

Below is the Python script I used for my first GPIO-based mini project. It blinks two LEDs connected to GPIO pins 17 and 27 in an alternating pattern. This project helped me understand GPIO output control, timing delays, and how to safely handle GPIO cleanup. 

            import RPi.GPIO as GPIO
            import time

            GPIO.setmode(GPIO.BCM)

            led1 = 17
            led2 = 27

            GPIO.setup(led1, GPIO.OUT)
            GPIO.setup(led2, GPIO.OUT)

            try:
                while True:
                    GPIO.output(led1, GPIO.HIGH)
                    GPIO.output(led2, GPIO.LOW)
                    time.sleep(1)
                    
                    GPIO.output(led1, GPIO.LOW)
                    GPIO.output(led2, GPIO.HIGH)
                    time.sleep(1)
            except KeyboardInterrupt:
                GPIO.cleanup()

- I imported RPi.GPIO to control the Pi‚Äôs GPIO pins and time for delay. - I used GPIO.setmode(GPIO.BCM) to use BCM pin numbering. - Pins 17 and 27 were defined as outputs and connected to the two LEDs. - Inside a try block, I created an infinite while True loop. - Each loop cycle turns LED 1 ON and LED 2 OFF, waits 1 second, then reverses it ‚Äî creating a ‚Äúping-pong‚Äù effect. - The except KeyboardInterrupt ensures that when I stop the script using Ctrl+C, the GPIO.cleanup() resets the pins safely.
